# 3. 데이터 및 경로 명세

## 3.1. 입력 데이터

### 3.1.1. 에어테이블 행 데이터
에어테이블에서 행을 선택 후 `Ctrl+C`로 복사하면 탭(`\t`)으로 구분된 텍스트가 클립보드에 저장됩니다.

**예시 원본 텍스트:**
```
13431	2026-03-30 16:30	4. 결제·발주·납품	오청인	기존 테스트	문정수	부품 2종
```

**파싱 대상 컬럼:**

| 인덱스 | 변수명 | 설명 | 예시 값 | 변환 규칙 |
|---|---|---|---|---|
| 1 | `COL_DATE` | 의뢰 등록 날짜 | `2026-03-30 16:30` | 앞 10자리 → `YYMMDD` (260330) |
| 5 | `COL_COMPANY` | 업체명 | `(주)이오에스티` | 그대로 사용 |
| 8 | `COL_PRODUCT` | 아이템명/제품명 | `부품 2종` | 그대로 사용 |

> [!warning] 컬럼 인덱스 주의
> 에어테이블 테이블 구조가 변경되면 `app.py` 상단의 `COL_DATE`, `COL_COMPANY`, `COL_PRODUCT` 값을 수정해야 합니다.

| `.pdf` | 📕 | 문서 | 파랑 (#dbeafe) |
| `.xlsx` `.xls` | 📗 | 스프레드시트 | 파랑 (#dbeafe) |

## 3.2. 출력 데이터

### 3.2.1. 생성되는 폴더
**폴더명 형식:** `{YYMMDD}_{아이템명}_{업체명}`

**예시:**
```
260330_부품 2종_(주)이오에스티
```

### 3.2.2. 복사되는 파일
- 원본 파일의 메타데이터(수정일 등)를 보존하여 복사 (`shutil.copy2`)
- 파일명은 원본 그대로 유지
- 동일 파일명 존재 시 덮어쓰기

## 3.3. 저장 경로

### 3.3.1. 3D 파일 탭
| 항목 | 값 |
|---|---|
| 변수명 | `TARGET_3D` |
| 전체 경로 | `Z:\2 제품개발팀\【1】 3D 의뢰파일\` |
| 최종 폴더 | `Z:\2 제품개발팀\【1】 3D 의뢰파일\{YYMMDD}_{아이템명}_{업체명}\` |
| 연도 구분 | 없음 (단일 폴더) |

### 3.3.2. 이미지 탭
| 항목 | 값 |
|---|---|
| 변수명 | `TARGET_IMAGE_BASE` |
| 기본 경로 | `Z:\1 공통 운영\【09】 아카이브 (촬영물 · 백업)\01 사진·영상 자료\01. 의뢰·제작 제품\` |
| 연도 폴더 | `{현재연도}년` (예: `2026년`, `2027년`) — 자동 생성 |
| 최종 폴더 | `...의뢰·제작 제품\2026년\{YYMMDD}_{아이템명}_{업체명}\` |

> [!info] 연도 자동 반영
> `get_image_target()` 함수가 프로그램 실행 시점의 연도를 기준으로 경로를 생성합니다. 연도가 바뀌면 코드 수정 없이 자동 반영됩니다.

## 3.4. 설정값 (하드코딩)

| 변수 | 파일 | 값 | 설명 |
|---|---|---|---|
| `COL_DATE` | app.py | `1` | 에어테이블 날짜 컬럼 인덱스 |
| `COL_COMPANY` | app.py | `5` | 에어테이블 업체명 컬럼 인덱스 |
| `COL_PRODUCT` | app.py | `8` | 에어테이블 아이템명 컬럼 인덱스 |
| `TARGET_3D` | utils.py | `Z:\2 제품개발팀\...` | 3D 파일 저장 기본 경로 |
| `TARGET_IMAGE_BASE` | utils.py | `Z:\1 공통 운영\...` | 이미지 저장 기본 경로 |
